Reports var properties with default setter and getter that doesn't reference backing field.

A var property with default setter and getter that doesn't reference backing field have no effect on calling its setter, therefore it will be more clear to change such property to val and delete the initializer.

'Change to val and delete initializer' quick-fix can be used to amend the code automatically.

Example:


// This property always returns '1' and it doesn't important that the property is a 'var'
var foo: Int = 0
    get() = 1
